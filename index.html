NoCSS

Build any design,
without writing CSS.

Write HTML intuitively with classes like color:red, textAlign:center and display:flex to build any design directly in your markup, while we take care of the CSS for you!

npm install @dcodestudios/nocss

Insanely Fast and Furious!

I was forever looking for ways to make CSS scalable without having to go through a build process over and over again, and with minimal back-and-forth movement between files. With NoCSS, you can just add your rules on the fly, making the development process extremely scalable, ridiculously fast, and most importantly, enjoyable :)

Awais Kazmi Creator of nocss

Speed

Build as you go.

Need to add 24px font size to an element? Simply give it the class fs:24px and you're done! Try giving it a class of color:red to change its color.

Learn more

Example:

<h1 class="fs:24px color:red"><h1/>

No Constraints

The entire CSS universe,
at your disposal

(Almost) All the CSS properties are supported, including and not limited to modifying an element's display, colors, spacing, typography, shadows, and everything else that you require to design your awesome looking design.

Learn more

Example:

<div class="d:flex p:24px"><div/>

Performance

It's tiny af.

NoCSS only generates CSS from the rules defined in your markup, which means your final bundle will never have any extra code that you're never going to use, which ususally comes with other frameworks.

Mobile-First

Everything is Responsive.

Writing and managing complex media queries is no fun at all. NoCSS allows you to build responsive designs right in your HTML instead.

Just add a screen size in front of any class and watch it magically apply at a specific breakpoint.

Learn more

Example:

<div class="fs:14px md:fs:24px"><div/>

States

Hover, focus, and active states also covered.

Simply prepend any of the states mentioned above before any class to add state-based styles an element.

Learn more

Example:

<p class="color:red hover:color:blue"><p/>

Responsive States

You can add responsive state-based styles just as easily!

If somehow you want to add different hover effects for different screen sizes, you have the power to do so as well.

Learn more

Example:

<p class="hover:color:red hover:md:color:blue"><p/>

Components

Duplication? Way ahead of you :)

If you're repeating the same utilities over and over again, simply add a componet attribute and boom — you've got a single source of truth so you can make changes in one place.

Learn more

Example:

<div component="button" class="p:24px fs:14px"><div/>

Use the component's name as a class to declare an instance:

<div class="button"><div/>

Overrides

Want to override a component? You got it!

You can override all the properties of a component's instance if it is absolutely necessary. This proves especially handy if you just want to modify a single property and don't want to create an entirely new component just for that one change.

Learn more

Example:

<div class="button borderRadius:100px"><div/>
